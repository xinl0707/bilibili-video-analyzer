# B 站视频快速分析 — Edge 扩展开发计划 v2

## 一、项目概述

制作一个全新的 Edge 浏览器扩展。一键点击即可自动获取 B 站视频字幕或音频，通过 AI 语音识别和大模型生成视频摘要，并在侧边栏中提供对话交互、多格式导出等功能。

### 核心卖点
- **智能获取文字**：优先使用 B 站 CC 字幕（免费快速），无字幕时自动降级为音频 ASR 识别
- **一键操作**：点击图标或快捷键，全自动完成
- **边看边聊**：侧边栏内与 AI 对话深入分析视频内容
- **后台运行**：分析过程不阻塞浏览，可切换标签页

## 二、核心技术栈

| 技术 | 用途 |
|---|---|
| Manifest V3 | Edge 扩展标准 |
| Service Worker | 后台脚本（资源拦截、API 调用、转码调度） |
| Content Script | 注入 B 站页面，拦截 playurl API、获取字幕 |
| Side Panel | 侧边栏 UI（进度展示、摘要、对话交互） |
| mux.js | .m4s 分片 MP4 提取 AAC 音频 |
| lamejs | PCM → MP3 编码 |
| chrome.storage.local | 配置存储、历史记录、性能数据 |

## 四、核心流程（含字幕兜底）

```
用户点击扩展图标（或按快捷键）
    ↓
① background service-worker 激活
    ↓
② 注入 content/interceptor.js 到当前 B 站标签页
    ↓
③ 获取视频信息（标题、UP主、BV号、CID）
    ↓
④ 尝试获取 B 站 CC 字幕
   ├─ 有字幕 → 直接使用字幕文本 → 跳到 ⑨
   └─ 无字幕 ↓
⑤ 拦截 playurl API 响应，提取 audio[] 中 bandwidth 最大的音频 URL
    ↓
⑥ 下载 .m4s 音频文件（带 referer/cookie 请求头）
    ↓
⑦ 音频转码：.m4s → mux.js 提取 AAC → AudioContext 解码 → lamejs 编码 MP3
    ↓
⑧ 智能分片（若 >10MB）→ 逐片 ASR 识别 → 拼接文字
    ↓
⑨ 将完整文字发送给大模型生成摘要（流式返回）
    ↓
⑩ 侧边栏展示：视频信息 + 摘要 + 对话区
    ↓
⑪ 用户可继续对话 / 导出 / 一键复制
```

## 六、关键技术细节

### 6.1 B 站字幕获取（utils/subtitle-fetcher.js）

B 站视频可能有 CC 字幕，通过以下 API 获取：
```
GET https://api.bilibili.com/x/player/v2?bvid={bvid}&cid={cid}
```
响应中 `data.subtitle.subtitles` 数组包含字幕信息：
```json
{
    "data": {
        "subtitle": {
            "subtitles": [
                {
                    "lan": "zh-CN",
                    "subtitle_url": "//aisubtitle.hdslb.com/bfs/ai_subtitle/xxx.json"
                }
            ]
        }
    }
}
```
字幕 JSON 文件包含时间戳和文字：
```json
{
    "body": [
        { "from": 0.0, "to": 2.5, "content": "你好" },
        { "from": 2.5, "to": 5.0, "content": "今天我们来聊聊" }
    ]
}
```

**策略**：
1. 先请求 player/v2 API 检查是否有字幕
2. 有字幕 → 下载字幕 JSON → 拼接为完整文字 → 跳过 ASR
3. 无字幕 → 走音频 ASR 流程

### 6.2 B 站音频拦截（content/interceptor.js）

**目标**：拦截 `api.bilibili.com/x/player/wbi/playurl` 响应，提取音频 URL。

**方案**：hook `fetch` 和 `XMLHttpRequest`，检测响应 URL 是否匹配 playurl API。

```javascript
// 伪代码
const originalFetch = window.fetch;
window.fetch = async function(...args) {
    const response = await originalFetch.apply(this, args);
    const url = typeof args[0] === 'string' ? args[0] : args[0].url;

    if (url.includes('/x/player/playurl') || url.includes('/x/player/wbi/playurl')) {
        const cloned = response.clone();
        const json = await cloned.json();
        if (json.data?.dash?.audio?.length) {
            const bestAudio = json.data.dash.audio
                .sort((a, b) => b.bandwidth - a.bandwidth)[0];
            chrome.runtime.sendMessage({
                type: 'AUDIO_FOUND',
                url: bestAudio.baseUrl,
                backupUrl: bestAudio.backupUrl?.[0],
                bandwidth: bestAudio.bandwidth,
                codecs: bestAudio.codecs
            });
        }
    }
    return response;
};
```

**注意**：
- 需要同时 hook `XMLHttpRequest`（B 站部分请求用 XHR）
- 音频 URL 需要带正确的请求头（referer、cookie）才能下载
- 用户必须先登录 B 站（cookie 鉴权）

```javascript
function calculateChunks(mp3Size) {
    const MAX_SIZE = 10 * 1024 * 1024; // 10MB
    if (mp3Size <= MAX_SIZE) return 1;
    return Math.ceil(mp3Size / MAX_SIZE) + 1; // 向上取整 + 1 保底
```

### 6.6 进度展示与时间预估

**进度计算**：
- 有字幕时：字幕获取 = 100%（秒级完成）
- 无字幕时：
  - 下载+转码：占 20%
  - ASR 识别：占 80%，按已处理片段/总片段比例

**时间预估（历史学习）**：
```javascript
// 每次识别完成后记录
const history = await storage.getAsrHistory();
history.push({
    audioSizeMB: mp3Size / 1024 / 1024,
    durationSec: elapsedSeconds,
    timestamp: Date.now()
});
if (history.length > 20) history.shift();
await storage.setAsrHistory(history);

// 预估时
function estimateRemaining(audioSizeMB, processedMB) {
    const history = await storage.getAsrHistory();
    if (history.length === 0) return null; // 首次无法预估
    const avgSpeed = history.reduce((sum, h) => sum + h.audioSizeMB / h.durationSec, 0) / history.length;
    return (audioSizeMB - processedMB) / avgSpeed;
}
```
